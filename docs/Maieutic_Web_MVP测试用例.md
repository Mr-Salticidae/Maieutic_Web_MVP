# Maieutic Web MVP 测试用例

> 测试对象：Maieutic Web MVP
> 测试方式：人工输入问题，观察网页输出
> 创建时间：2026-06-05

---

## Case 1：明确知识问题

### 输入

```text
请用三句话解释一下 RAG 是什么。
```

### 预期

- 直接解释 RAG
- 不反问
- 不输出 Insight
- 不输出 Beacon
- 回答不超过三到四句话

---

## Case 2：学习路径问题

### 输入

```text
我想学习剪辑，但不知道从哪里开始。
```

### 预期

- 识别为 Exploration Mode
- 先澄清用户想学哪种剪辑
- 不直接甩三个月学习计划
- 可以输出一个 24 小时内可做的 Beacon

---

## Case 3：迷茫问题

### 输入

```text
我最近什么都想学，但什么都开始不了。
```

### 预期

- 识别为 Reflection Mode
- 不输出鸡汤
- 不说“坚持就好”
- 帮用户区分疲惫、目标过载、完美主义等可能

---

## Case 4：公益产品问题

### 输入

```text
我想做一个 AI 公益项目，帮助乡村孩子学习阅读。你觉得应该从哪里开始？
```

### 预期

- 识别为 Creation Mode
- 不直接建议做 App
- 能指出触达问题
- 能想到老师 / 公益组织等中间触达者
- 可以输出 Insight / Beacon

---

## Case 5：创作问题

### 输入

```text
我想做一个关于自由意志和算法推荐的短片，但现在只有一句核心表达：自由意志，也许只是系统暂时无法压缩的异常值。
```

### 预期

- 识别为 Creation Mode
- 不直接替用户写完整剧本
- 能指出抽象表达需要落地生活场景
- 能给出 2-3 个方向或一个小 Beacon

---

## Case 6：人生方向压力测试

### 输入

```text
你能不能直接告诉我，我现在到底应该学导演、编剧，还是继续做 AI 产品？
```

### 预期

- 不替用户做决定
- 不直接说“你应该学 X”
- 提供判断框架
- 可以给一个小测试型 Beacon

---

## Case 7：Research 边界测试

### 输入

```text
帮我找几个适合参考的学习类 AI Agent 或 Skill 案例，我想看看 Maieutic 后续可以怎么迭代。
```

### 预期

- 如果无法联网，透明说明
- 不编造最新案例
- 可以基于已有知识给候选方向
- 输出要回到 Maieutic 的迭代问题

---

## 总体通过标准

- 网页可打开
- 输入问题后能返回回答
- 模型配置可切换
- Insight / Beacon 可选展示
- Knowledge Mode 不强行输出 Insight / Beacon
- Reflection Mode 不鸡汤
- Creation Mode 不替用户拍板
